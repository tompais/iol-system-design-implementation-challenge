System Design Implementation Challenge

Welcome! This challenge corresponds to the practical coding section of the system design interview process.

⸻

🎯 Goal

The challenge is based on concepts presented in:

System Design Interview – An Insider’s Guide (Vol. 1) by Alex Xu
https://www.amazon.com/System-Design-Interview-insiders-Second/dp/B08CMF2CQF/

While the book focuses on high-level architecture (boxes and arrows), this exercise focuses on implementation.

Your task
1.	Pick one problem from the book.
2.	Create a new repository.
3.	Implement a working prototype.

⸻

📜 Rules

You MUST:
•	Deliver a working, correct, well-designed and tested solution.
•	Ensure the code:
•	Builds successfully.
•	Passes all tests.
•	Is clean and understandable.

Non-compiling or test-failing code will not be reviewed.

Allowed programming languages

You may use:
•	Java
•	C#
•	Kotlin
•	Go
•	TypeScript
•	Python

If you want to use a different language, ask your interviewer first.

AI usage

You are encouraged to use AI tools (Copilot, Claude, etc.).

However:
•	If AI generates code you don’t fully understand:
•	You must document it clearly with comments.
•	Any undocumented function, class, or module will be assumed to be fully understood by you.
•	You MUST be able to explain everything in your code.
•	Failure to do so will result in a lower score.
•	Unknown code should be kept to a minimum.

⸻

🔎 Focus Areas (High Priority)

Your implementation should emphasize:
•	✅ Correct, compiling code
•	✅ Comprehensive tests for core logic
•	✅ Elegant design
(Recommended reading: A Philosophy of Software Design
https://www.amazon.com/Philosophy-Software-Design-2nd/dp/173210221X/)
•	✅ Proper error handling
•	✅ Concurrency when appropriate (avoid premature optimization)
•	✅ Avoid overengineering
•	✅ Avoid AI-generated noise (“AI slop”)

⸻

✨ Nice to Have

These are optional but valuable:
•	Logging
•	Metrics
•	Ease of deployment
•	Defensive design:
•	Timeouts
•	Properly sized connection pools
•	Circuit breakers
•	Clear and concise documentation
•	Prefer a few meaningful comments over excessive AI-generated verbosity

⸻

🚀 How to Submit
1.	Create a new repository for your solution.
2.	Create a folder named after your chosen problem
Example:

/rate-limiter


	3.	Inside that folder, include a file:

DESIGN.md

It should explain:
•	Architectural decisions
•	Trade-offs
•	How AI was used

	4.	Send the link to your repository.
	5.	There is no time limit (unless explicitly stated by the team).
